Project Overview

ObjectReact is a vision-based navigation framework that enables an agent to navigate towards a goal image using object-centric representations called WayObject Costmaps. The original implementation provides a complete pipeline for generating training data in HM3D environments and training the navigation controller.

The objective of this project is to extend the dataset generation pipeline to Gibson environments. The resulting pipeline generates navigation trajectories, topological object graphs, topometric object graphs, WayObject Costmaps, and the complete training dataset required by the original ObjectReact controller.

This website documents the complete workflow, implementation details, repository modifications, common implementation issues, debugging strategies, and the steps required to reproduce the entire pipeline for future researchers and interns.

Workflow Overview

The complete workflow developed during this project consists of five major stages. Each stage produces the inputs required by the next stage and together forms the complete data generation and training pipeline for ObjectReact.

  1. Environment Setup
    Install Habitat-Lab, Habitat-Sim and project dependencies. Download the Gibson scenes and verify simulator rendering.

  2. Dataset Generation
    Generate trajectories inside Gibson environments and save RGB images, agent states and trajectory metadata.

  3. Topometric Map Construction
    Segment objects using FastSAM, associate them using LightGlue, integrate simulator depth and generate topometric graphs together with WayObject Costmaps.

  4. Controller Training
    Train the original ObjectReact controller using the generated Gibson dataset and monitor convergence using Weights & Biases.

  5. Controller Evaluation
    Evaluate the trained controller inside Habitat and visualize navigation behaviour, controller predictions and WayObject Costmaps.

Dataset Generation Pipeline

The primary objective of this project is to generate training data for the ObjectReact navigation framework using Gibson environments. The generated dataset contains navigation trajectories, object-centric graphs, WayObject Costmaps, and all supporting metadata required for controller training.

The complete pipeline consists of six major stages. Each stage consumes artifacts generated by the previous stage and produces inputs for the following stage.

  1. Trajectory Generation
  2. Topological Graph Generation
  3. Topometric Graph Generation
  4. WayObject Costmap Generation
  5. Dataset Preparation
  6. Controller Training

The following sections describe every stage in detail, including the scripts involved, generated files, required modifications, debugging strategies and verification steps.

Stage 2: Topological Graph Generation

Objective

Construct an object-centric topological graph from the RGB observations generated in Stage 1. Objects are segmented using FastSAM and associated across consecutive frames using LightGlue.

Inputs

  • RGB images
  • FastSAM
  • LightGlue

Outputs

  • nodes_fast_sam_graphObject_4_lightglue.pickle

Purpose

Represents the scene as an object graph without geometric depth information.

Status

✔ Completed successfully for Gibson environments.

Stage 3: Topometric Graph Generation

Objective

Extend the topological graph by integrating simulator-rendered depth information to obtain a topometric object graph.

Inputs

  • Topological graph
  • Habitat simulator
  • Depth observations

Outputs

  • nodes_fast_sam_graphObject_4_lightglue_depth_sim.pickle

Purpose

Stores three-dimensional relationships between segmented objects.

Status

✔ Completed successfully for Gibson environments.

Stage 4: WayObject Costmap Generation

Objective

Compute shortest path lengths over the topometric graph and encode them as WayObject Costmaps for every observation.

Inputs

  • Topometric graph
  • Shortest-path computation

Outputs

  • gibson_gt_topometric_e3d_masks_pls.h5

Purpose

Generates the object-centric navigation representation used by the ObjectReact controller.

Status

✔ Completed successfully for Gibson environments.

Stage 5: Dataset Preparation

Objective

Prepare the generated Gibson dataset in the format expected by the ObjectReact training pipeline.

Outputs

  • Train/Test trajectory splits
  • Configuration files
  • HDF5 costmaps

Purpose

Organizes the generated data into a structure compatible with the original ObjectReact implementation.

Status

✔ Completed successfully.

Stage 6: Controller Training

Objective

Train the original ObjectReact navigation controller using the generated Gibson dataset and WayObject Costmaps.

Inputs

  • Trajectory dataset
  • WayObject Costmaps
  • Training configuration

Outputs

  • Trained controller checkpoints
  • Weights & Biases logs

Purpose

Learn an action policy that predicts navigation actions from RGB observations and WayObject Costmaps.

Status

✔ Successfully trained on the generated Gibson dataset.

Stage 7: Controller Evaluation

Objective

Evaluate the trained controller inside Habitat and analyse navigation behaviour both quantitatively and qualitatively.

Outputs

  • Navigation trajectories
  • Qualitative visualizations
  • Evaluation metrics

Purpose

Verify that the controller generalizes to Gibson environments using the generated training dataset.

Status

⏳ Evaluation pipeline under development.
