Project Overview

ObjectReact is a vision-based navigation framework that enables an agent to navigate towards a goal image using object-centric representations called WayObject Costmaps. The original implementation provides a complete pipeline for generating training data in HM3D environments and training the navigation controller.

The objective of this project is to extend the dataset generation pipeline to Gibson environments. The resulting pipeline generates navigation trajectories, topological object graphs, topometric object graphs, WayObject Costmaps, and the complete training dataset required by the original ObjectReact controller.

This website documents the complete workflow, implementation details, repository modifications, common implementation issues, debugging strategies, and the steps required to reproduce the entire pipeline for future researchers and interns.

Workflow Overview

The complete workflow developed during this project consists of five major stages. Each stage produces the inputs required by the next stage and together forms the complete data generation and training pipeline for ObjectReact.

  1. Environment Setup
    Install Habitat-Lab, Habitat-Sim and project dependencies. Download the Gibson scenes and verify simulator rendering.

  2. Dataset Generation
    Generate trajectories inside Gibson environments and save RGB images, agent states and trajectory metadata.

  3. Topometric Map Construction
    Segment objects using FastSAM, associate them using LightGlue, integrate simulator depth and generate topometric graphs together with WayObject Costmaps.

  4. Controller Training
    Train the original ObjectReact controller using the generated Gibson dataset and monitor convergence using Weights & Biases.

  5. Controller Evaluation
    Evaluate the trained controller inside Habitat and visualize navigation behaviour, controller predictions and WayObject Costmaps.

Dataset Generation Pipeline

The primary objective of this project is to generate training data for the ObjectReact navigation framework using Gibson environments. The generated dataset contains navigation trajectories, object-centric graphs, WayObject Costmaps, and all supporting metadata required for controller training.

The complete pipeline consists of six major stages. Each stage consumes artifacts generated by the previous stage and produces inputs for the following stage.

  1. Trajectory Generation
  2. Topological Graph Generation
  3. Topometric Graph Generation
  4. WayObject Costmap Generation
  5. Dataset Preparation
  6. Controller Training
